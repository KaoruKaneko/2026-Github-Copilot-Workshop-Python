# Pomodoro Timer 実装機能一覧

このドキュメントは、ポモドーロタイマーWebアプリで実装が必要な機能を整理したものです。

## 1. MVP（最小実用）

### 1.1 タイマー基本操作
- 開始（Start）
- 一時停止（Pause）
- 再開（Resume）
- リセット（Reset）
- スキップ（Skip）

### 1.2 セッションモード
- Focusモード
- Short Breakモード
- Long Breakモード
- モードの手動切替

### 1.3 タイマー表示
- 残り時間（mm:ss）表示
- 現在モード表示
- 0秒到達時の完了処理

### 1.4 ポモドーロサイクル管理
- Focus完了回数のカウント
- 指定回数ごとのLong Breakへの遷移

### 1.5 設定管理
- 作業時間（work_minutes）
- 短休憩時間（short_break_minutes）
- 長休憩時間（long_break_minutes）
- 長休憩までの回数（long_break_interval）

### 1.6 設定の永続化
- 設定値の保存
- 再読み込み後の設定復元

### 1.7 セッション履歴
- セッション結果の保存（完了/中断）
- セッション種別、開始時刻、終了時刻、実績秒数の記録

### 1.8 基本統計
- 当日の完了セッション数
- 当日の集中時間合計

### 1.9 API連携
- 設定取得API
- 設定更新API
- セッション保存API
- 統計取得API

### 1.10 基本UI
- タイマー表示エリア
- 操作ボタン
- モード表示/切替UI
- 設定UI
- 統計表示UI

## 2. 準MVP（品質向上のため早期導入推奨）

### 2.1 タイマー精度維持
- 終了予定時刻との差分で残り時間を計算
- タブ非アクティブ復帰時のズレ補正

### 2.2 状態遷移の厳密化
- 不正遷移の防止（例: 停止中のみ再開可）
- 状態とイベントに基づく遷移制御

### 2.3 入力バリデーション
- 設定値の範囲チェック
- 型チェック
- 不正値入力時のエラーメッセージ表示

### 2.4 エラーハンドリング
- API失敗時のユーザー通知
- 保存失敗時の再試行またはフォールバック

### 2.5 レスポンシブ対応
- モバイルとデスクトップでのレイアウト最適化

### 2.6 アクセシビリティ
- キーボード操作対応
- フォーカス可視化
- 可読性の高いコントラスト

### 2.7 通知機能
- セッション終了時の通知音
- （任意）ブラウザ通知
- 通知ON/OFF設定

## 3. テスト容易性のために実装必須な設計機能

### 3.1 状態遷移ロジック分離
- UIから独立した純粋ロジックとして実装

### 3.2 Clock抽象
- 現在時刻取得を抽象化（clock.now）
- テスト時に時刻固定を可能にする

### 3.3 Repository抽象
- SettingsRepository / SessionRepository のインターフェース化
- 本番（File/LocalStorage）とテスト（InMemory）実装の分離

### 3.4 API契約固定
- APIレスポンスのJSON構造を明文化
- 契約テストで互換性を維持

### 3.5 境界ケース対応
- 残り時間0秒到達境界
- pause/resume連続操作
- Long Break切替境界
- 設定最小値/最大値/不正値

## 4. 将来拡張候補（任意）
- タスク管理との連携
- 週次/月次の詳細分析
- ユーザー認証
- マルチデバイス同期
- 外部通知連携（Slack等）

2. 履歴・統計・API連携（1.7〜1.9）
